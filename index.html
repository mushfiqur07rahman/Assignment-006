Frequently Asked Questions

1. The difference between var, let, and const.
`var` can be redeclared and used before it's defined, which can cause issues. `let` allows changes but cannot be redeclared, and `const` cannot be changed or redeclared after it's assigned.
2. The difference between map(), forEach(), and filter().
`map()` creates a new array by applying a function to each element, `forEach()` simply loops through each element without returning anything, and `filter()` creates a new array by selecting elements that meet a condition.
3. Explain arrow functions and how they are different from regular functions.
Arrow functions provide a shorter syntax and don't have their own `this` value, unlike regular functions that do. Regular functions can also be used as methods with a `this` context, while arrow functions can't.
4. How JavaScript Promises work?
JavaScript Promises represent a value that may not be available yet, allowing asynchronous code to run without blocking. They can be in one of three states: pending, resolved, or rejected, and handle results with `.then()` for success or `.catch()` for errors.
5. How closures work in JavaScript?
Closures allow a function to remember and access variables from its outer scope even after the outer function has finished executing. This happens because the inner function maintains a reference to the outer function's variables.
